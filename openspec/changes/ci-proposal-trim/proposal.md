# ci-proposal-trim

## Why

Proposals generated by AI tend to be over-scoped. Reviewing and cutting them requires manual file editing. We need a way to trim a proposal by leaving review comments directly on the PR — the pipeline applies the changes automatically, one comment at a time.

## What Changes

- New GitHub Actions workflow: `.github/workflows/ci-proposal-trim.yml`
- Triggered by PR review comments on `proposal/**` branches
- Claude reads the comment + surrounding diff context + full `openspec/changes/<name>/` directory
- Applies changes to proposal artifacts (spec files, tasks.md, proposal.md, design.md)
- Commits result to the proposal branch
- Marks the comment as resolved (👍 reaction)

## Capabilities

### New Capabilities

- `ci-proposal-trim`: Workflow that applies a single PR review comment as an edit to the proposal artifacts

## Workflow

```
pr_review_comment.created
        │
        ├── guard: branch matches proposal/**
        ├── guard: author_association == 'OWNER'
        │
        ▼
   parse branch → extract <name>
        │
        ▼
   checkout proposal branch
        │
        ▼
   Claude receives:
     - comment.body
     - comment.path
     - comment.diff_hunk  (line context)
     - openspec/changes/<name>/**
        │
        ▼
   Claude edits openspec/changes/<name>/**
     (delete specs, rewrite, add, update tasks/proposal)
        │
        ▼
   git commit + push → proposal branch
        │
        ▼
   add 👍 reaction to comment (marks it resolved)

CONCURRENCY: group per branch, cancel-in-progress: false (queue)
```
